# Contributing to the Project

We welcome contributions from the community! Whether you're fixing bugs, adding new features, or improving documentation, your help is appreciated. Please follow the guidelines below to ensure a smooth contribution process.

## How to Contribute

1. **Fork the Repository**: Start by forking the repository to your own GitHub account.
2. **Clone the Fork**: Clone your forked repository to your local machine.
3. **Create a Branch**: Create a new branch for your feature or bug fix.
4. **Make Changes**: Implement your changes in the new branch.
5. **Commit Changes**: Commit your changes with clear and descriptive messages.
6. **Push Changes**: Push your changes to your forked repository on GitHub.
7. **Open a Pull Request**: Open a pull request against the main repository, describing the changes you made and why they are necessary.
8. **Respond to Feedback**: Be prepared to discuss and revise your pull request based on feedback from the maintainers.
9. **Celebrate**: Once your pull request is merged, celebrate your contribution to the project!
10. **Code of Conduct**: Please adhere to the project's [Code of Conduct](CODE_OF_CONDUCT.md) while contributing.


## Code of Conduct

Please, refer to the [Code of Conduct](CODE_OF_CONDUCT.md) for guidelines on expected behavior and how to report issues. We are committed to providing a welcoming and inclusive environment for all contributors.

## Where to do we need help

We are looking for help in the following areas:
- Improving and adding new profiles of teachers.
- Fixing typos and grammatical errors in the documentation or in the prompts.
- Adding more info about possible LLMs integrations and usage examples.
- Adding crazy ideas as issues or discussions to explore new possibilities with the project (for example, use local LLMs models or API based ones).
